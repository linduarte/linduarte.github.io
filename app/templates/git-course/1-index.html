O que é 'Git' e por que ele é importante?

É um sistema de controle de versão distribuída, permitindo que vários desenvolvedores colaborem em um projeto simultaneamente.
O Git acompanha as alterações realizadas nos códigos-fonte, permite o versionamento e facilita a colaboração entre os membros de um grupo.
O Git tornou-se importante porque é robusto, flexível e eficiente no controle de alterações em ambientes colaborativos de desenvolvimento.
Sua ampla adoção, funcionalidades poderosas e suporte a diversos fluxos de trabalho fazem dele uma ferramenta essencial nas práticas modernas de desenvolvimento.
Em pequenos projetos ou aplicações de larga escala, o Git, com suas várias funcionalidades, contribui diretamente para a qualidade e rastreabilidade do software.

Principais estados dos arquivos no Git

O Git possui três estados nos quais seus arquivos podem ser armazenados. Estes estados correspondem a estágios do projeto: "modificado", "staged" e "comitado".
"Modificado" – houve uma alteração no arquivo, mas ele ainda não foi incluído no próximo commit.
"Staged" – você marcou o arquivo modificado para ser incluído no próximo commit.
"Comitado" – os dados referentes ao arquivo estão registrados com segurança no repositório.

Conceitos chave: 'Repositories', 'Commits' e 'Branches'
  • 'Repositories': São recipientes do projeto; armazenam arquivos e seu histórico completo.
  • 'Commits': Cada commit representa um instante (snapshot) no tempo, permitindo voltar atrás a estados anteriores.
  • 'Branches': Permitem desenvolvimento paralelo para novas funcionalidades ou correções independentes.
Git workflow (fluxo de trabalho)
  • Inicializando um repositório: git init
  • Adicionando arquivos à área de staging: git add
  • Realizando um commit: git commit -m "Mensagem do commit"
  • Verificando o status: git status
  • Vendo o histórico de commits: git log
Terminal Git e por que não uma interface gráfica

A interface gráfica pode ser intuitiva, mas o terminal oferece vantagens decisivas: maior controle e visibilidade textual das mudanças, dos branches e de todo o histórico.
Essa transparência é crucial para entender a complexidade do versionamento e agir rapidamente.
O terminal também permite a criação de scripts para automação de tarefas repetitivas ou fluxos personalizados.

E como não falar de GitHub?

O engenheiro de software Tom Preston-Werner, percebendo a necessidade de uma forma mais eficiente de compartilhar e gerenciar código entre desenvolvedores, iniciou o GitHub em 2008. Com apoio de investimentos, cresceu rapidamente e em dois anos já tinha um milhão de usuários. Em 2018 foi adquirido pela Microsoft por 7,5 bilhões de dólares.
O GitHub é um serviço de hospedagem para desenvolvimento de software baseado em Git, oferecendo todas as funcionalidades de controle distribuído de versão e adicionando recursos próprios (integração, revisão de código, ações automatizadas, etc.).

Sobre o Git

Com mais de 57.000 alterações comitadas e mais de 1.300 colaboradores, o Git tornou-se a ferramenta de controle de versões mais usada no mundo.
Seu criador é Linus Torvalds, também criador do kernel do Linux. O Git nasceu em 2005 no contexto do desenvolvimento desse kernel.
Desde então, o mantenedor principal, Junio Hamano, coordena a evolução do projeto com a ampla comunidade open source.

Por que se chama Git?

'Git' é uma gíria do inglês britânico que pode significar 'pessoa desagradável'. O nome foi escolhido com humor pelo próprio criador.

Por que Linus Torvalds criou o Git?

Durante o desenvolvimento do kernel do Linux, utilizava-se o BitKeeper. Quando o uso gratuito foi retirado, Linus criou o Git como alternativa livre e aberta.

Um pouco mais sobre Git

O Git rastreia alterações em arquivos e não serve apenas para código-fonte: pode apoiar controle de versões em documentação técnica, textos longos ou configurações complexas.

Uma vantagem do Git

O Git roda localmente. Não depende de hospedagem em nuvem para operações básicas. É autossuficiente.

Alternativas ao Git

Algumas alternativas incluem Azure DevOps Server, Helix Core, AWS CodeCommit, Subversion e Rational ClearCase (pagas). Outra opção relevante é o Mercurial, conhecido pela configuração simples.
