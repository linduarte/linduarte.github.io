O que é um 'branch' (ramo)?

Um 'branch' representa uma linha independente de desenvolvimento no projeto com sua própria história de comites. Voce pode listar os 'branches' no repositório local bastando usar o comando:

git branch
Quando se associa o parâmetro '--remote' ao comando 'git branch' é possível listar os 'branches' remotos. 
git branch --remote

Agora nosso repositório posui somente um único ramo, 'branch' chamado 'main'. O ramo 'main' é o ramo padrão do Git. Este ramo é criado automaticamente quando se inicia um novo repositório. O ramo 'main' é o ramo principal do projeto. Ou seja é deafult. Se compararmos o projeto Git como uma árvore o ramo 'main' seria o tronco da árvore. O ramo 'main' é o ramo que contém o código de produção. A partir do tronco é possivel criarmos linhas separadas de desenvolvimento como ramos idependentes com seus próprios comites.

Trabalhando com ramos (branches)

Uma ramificação no Git é uma linha de desenvolvimento independente com um histórico de confirmação separado. Em grandes projetos, cada novo recurso ou correção de bug geralmente é desenvolvido em uma ramificação separada e, uma vez concluído, mesclado na base de código principal. Neste tópico, examinaremos as operações de ramificação e mesclagem no Git.
