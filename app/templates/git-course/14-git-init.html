git init

O comando git init desempenha um papel crucial no universo do Git, um sistema de controle de versão distribuído amplamente utilizado para gerenciar código-fonte. Esse comando é utilizado para criar um novo repositório Git vazio ou para reinicializar um já existente. Basicamente, ele configura as estruturas necessárias em um diretório especificado para acompanhar e gerenciar as mudanças nos arquivos. Seja iniciando um novo projeto, migrando um projeto existente para um fluxo de trabalho Git ou configurando um repositório para colaboração, o git init é geralmente a primeira etapa do processo.

A imagem acima mostra a criação de dois diretórios para o 'projeto-a'. Ao acionar o comando git init o referido projeto passa a ser controlado pelo git. Observa-se ainda que aparece em destaque um símbolo de um 'ramo' e o nome dado a ele é 'main'. O ramo é mais conhecido tecnicamente como 'branch'. Desta forma temos a certeza que foi inicializado um repositório Git.

Definição

O git init cria um novo repositório Git vazio ou reinicializa um existente. É o primeiro comando usado quando você quer começar a versionar um projeto.

Iniciando um repositório com um nome específico para o ramo inicial

Por padrão, o Git cria um branch chamado 'main' (ou 'master' em versões antigas). Você pode especificar um nome diferente:

git init --initial-branch=desenvolvimento
git init -b meu-branch-principal
