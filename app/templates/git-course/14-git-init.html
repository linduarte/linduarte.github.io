git init

O comando git init desempenha um papel crucial no universo do Git, um sistema de controle de versão distribuído amplamente utilizado para gerenciar código-fonte. Esse comando é utilizado para criar um novo repositório Git vazio ou para reinicializar um já existente. Basicamente, ele configura as estruturas necessárias em um diretório especificado para acompanhar e gerenciar as mudanças nos arquivos. Seja iniciando um novo projeto, migrando um projeto existente para um fluxo de trabalho Git ou configurando um repositório para colaboração, o git init é geralmente a primeira etapa do processo.

A imagem acima mostra a criação de dois diretórios para o 'projeto-a'. Ao acionar o comando 'git init' o referido projeto passa a ser controlado pelo git. Observa-se ainda que aparece em destaque um símbolo de um 'ramo' e o nome dado a ele é 'main'. O ramo é mais conhecido tecnicamente como 'branch'. Desta forma temos a certeza que foi inicializado o repositório do projeto-a.

Inicializando um repositório com um nome específico para o ramo (branch) inicial

git init --initial-branch=branch_name

Os desenvolvedores geralmente preferem usar nomes específicos como main ou trunk para o branch principal de um repositório em vez do padrão master. Esse comando é útil quando você quer definir um nome personalizado para o branch inicial desde o começo, alinhando com as convenções de nomenclatura de branches da sua equipe ou organização.

--initial-branch=branch_name: Esse flag permite que você defina o nome do ramo inicial no seu repositório em vez do padrão master(pouco usado atualmente). É especialmente útil para se alinhar com padrões da indústria ou preferência pessoal.
