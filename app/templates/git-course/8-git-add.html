git add

Adicionando arquivos ao projeto

Antes de usar o comando git add, é importante ter arquivos novos no seu projeto. Este comando é usado para adicionar essas mudanças ao stage, preparando-as para o próximo commit.

Passos para adicionar arquivos ao repositório:

    1. Faça o download dos arquivos usando este link.
    2. Descompacte os arquivos e copie cole no site "simple-website".
    3. Verifique pelo Windows Explorer a estrutura agora completa do repositório.

Esta imagem mostra usando o comando git status que representa como os arquivos do repositório estão em relação ao Git, ou seja em vermelho, que os arquivos não estão sendo rastreados pelo git, "untracked", indica que se use o comando git add para mudar o status dos arquivos. Sinaliza mais embaixo dizendo que nada foi adicionado para a realização do "commit". Mais embaixo temos a estrurura to repositório mostrada com os arquivos e um pasta 'style'.

git add path/to/file

Esse comando é usado quando você deseja preparar um arquivo específico para um commit. Por exemplo, você pode estar trabalhando em vários arquivos, mas apenas um está pronto para ser confirmado. Ao preparar esse arquivo individualmente, você pode garantir que ele seja o único incluído no próximo commit, mantendo o histórico de commits limpo e organizado.

  • 'git add':O comando base para preparar arquivos para commit.
  • 'path/to/file':O caminho específico do arquivo que você deseja adicionar ao index
Nota: O comando git add . adiciona todos os arquivos. Use git add <arquivo> para adicionar arquivos específicos.

Na imagem anterior podemos notar a aplicação do comando 'git add' em duas situações - na primeira aplicado na pasta 'style' contendo 4 arquivos e no segundo comando 'git add' aplicado usando o curinga '*' para aplicar em todos os arquivos com extensão 'html'. Ao se aplicar o comando git status observa-se que todos os arquivos foram 'staged', ou seja prontos para receberem a confirmação via o comando git commit. Alguns arquivos que já tiverem passados pelo comando git add podem ser alterados ou seja modificados e assim estes deverão também ser alvo do comando git add para que eles mudem de status, ou fiquem na condição 'staged', prontos para serem comitados.
